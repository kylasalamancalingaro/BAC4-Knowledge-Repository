FoundationHow AI models work
AI basics (high-level)

AI predicts useful output from the context you provide.

You provide instructions and context, the model reasons over patterns, then generates a response you must review.

1. Input

  • Your goal
  • Instructions (prompt)
  • Files or data
  • Constraints

2. AI Processing

  • Text becomes tokens
  • Model finds patterns
  • Better context = better output
  • Noise weakens results

3. Output

  • Requirements draft
  • User stories
  • Analysis summary
  • Next-step plan
Better input creates better output

Critical skillHow to write good prompts
Prompt structure

Good prompts get good results.

Give AI the context it needs: role, goal, inputs, outputs, constraints, and validation.

// Reusable prompt structure
Role: Who should AI act as?
Goal: What are we trying to achieve?
Context: What files/data to use?
Output: What should it produce?
Constraints: What rules to follow?
Validation: How should it check itself?

Example: User Story Generation

Act as a Business Analyst creating user stories for a data analytics platform. Turn stakeholder interview notes into structured user stories. Use the attached interview transcript with the CFO and operations team. Create user stories in the format: As a [role], I want [capability], so that [benefit]. Include acceptance criteria for each. Keep stories atomic, testable, and business-focused. Do not invent technical implementation details. Flag any assumptions separately and ask questions for unclear requirements.
Prompt quality determines output quality

Hands-onReal BA task
Practice task

Let's practice with a real scenario.

Use AI to analyze stakeholder interviews and create a requirements summary with user stories.

The Task

Input: Stakeholder interview transcripts (CFO, operations manager, data team lead)

Goal: Extract business requirements, identify conflicts, and draft user stories

Output: Structured requirements document with prioritized user stories

Your Prompt Should Include

  • Role: Business Analyst
  • Goal: Extract and structure requirements
  • Context: Interview transcripts
  • Output: Requirements + user stories
  • Constraints: No technical assumptions
  • Validation: Flag conflicts and gaps

SummaryRemember these
Key takeaways

Remember these principles as you start using AI.

01

Start Small

Begin with automation tasks. Master those before moving to augmentation or agency.

02

Context Matters

Better prompts with clear context produce better results. Garbage in, garbage out.

03

Always Review

AI outputs require human validation. You own the final judgment, not the AI.

04

Iterate

Refine your prompts based on results. Learn what works for your specific use cases.

AI is a tool, not a replacement for BA judgment
